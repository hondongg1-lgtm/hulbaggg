# دليل منصة السوق التفاعلية

## نظرة عامة

تم تحويل المنصة بالكامل إلى نظام سوق تفاعلي يربط المستخدمين بالمحلات القريبة من خلال ألعاب وجوائز.

---

## الواجهات الثلاث

### 1. الصفحة الرئيسية (/)
- صفحة تسويقية
- تسجيل الأعمال
- تسجيل البقالات

### 2. واجهة المستخدم (?user=true)
```
http://localhost:5173/?user=true
```

**التدفق الكامل:**

1. **صفحة البداية** → اختيار اللغة
2. **التحقق بالجوال** → OTP
3. **صفحة عروض حيك** 🔥
   - عرض المحلات حسب الحي
   - فلترة بالموقع
   - محاولة واحدة يومياً لكل محل
4. **لعبة Tap-to-Reveal** 🎮
   - تأخير 2-3 ثواني للتشويق
   - إما تفوز أو تحصل على جائزة ترضية
5. **عرض النتيجة**
   - إذا فزت: كود QR + كود نصي
   - إذا لم تفز: خصم بسيط

### 3. لوحة المعلن (?admin=true)
```
http://localhost:5173/?admin=true
```

**الميزات:**
- إنشاء حملة جديدة
- لوحة تحكم مع إحصائيات:
  - عدد المسحات
  - مرات اللعب
  - الفائزين
  - الجوائز المستلمة
  - معدل الفوز
  - معدل الاستلام

## قاعدة البيانات

### الجداول الجديدة

#### `neighborhoods`
أحياء الأحساء للاستهداف الجغرافي:
- المبرز
- الهفوف
- المنيزلة
- العيون
- الجفر
- الطرف
- العمران

#### `user_profiles`
معلومات إضافية عن المستخدمين:
- phone
- preferred_neighborhood
- last_active

#### `game_attempts`
تتبع محاولات اللعب:
- user_id
- campaign_id
- played_at
- won (boolean)
- prize_id

#### `prize_claims`
أكواد استلام الجوائز:
- claim_code (8 أحرف)
- expires_at (48 ساعة)
- status (pending/redeemed/expired)

### الحقول الجديدة في campaigns

- `neighborhood_id` - الحي المستهدف
- `daily_attempts_per_user` - محاولة واحدة يومياً
- `total_prize_pool` - مجموع الجوائز
- `win_probability` - نسبة الفوز (%)
- `consolation_prize` - جائزة الترضية
- `consolation_discount` - قيمة الخصم

## دوال PostgreSQL المهمة

### `can_user_play_today(user_id, campaign_id)`
تحقق إذا كان المستخدم يمكنه اللعب اليوم.

### `generate_claim_code()`
ينشئ كود فريد من 8 أحرف.

### `play_game(user_id, campaign_id)`
الدالة الرئيسية للعبة:
1. تحقق من الحد اليومي
2. تحدد الفوز/الخسارة بناءً على الاحتمالية
3. تختار جائزة عشوائية
4. تنشئ كود استلام
5. تسجل المحاولة

**مثال:**
```sql
SELECT play_game(
  'user-uuid-here',
  'campaign-uuid-here'
);
```

**النتيجة:**
```json
{
  "success": true,
  "won": true,
  "claim_code": "A7B3C9D2",
  "prize_id": "prize-uuid"
}
```

## كيفية الاختبار

### الخطوة 1: إنشاء حساب معلن

1. افتح `?admin=true`
2. سجل حساب جديد من الصفحة الرئيسية أو سجل دخول
3. ستنتقل تلقائياً للوحة التحكم

### الخطوة 2: إنشاء حملة

1. اضغط "+ حملة جديدة"
2. املأ البيانات:
   - اسم المحل: "مطعم الذواق"
   - النوع: مطعم
   - الحي: المبرز
   - عدد الأكياس: 1000
   - الجائزة: "خصم 50% على الوجبة"
   - نسبة الفوز: 15%
3. اضغط "إطلاق الحملة"

### الخطوة 3: تجربة واجهة المستخدم

1. افتح `?user=true` في تبويب جديد
2. أدخل رقم جوال: `0501234567`
3. افتح Console (F12) للحصول على OTP
4. أدخل OTP
5. ستظهر صفحة "عروض حيك" مع المحلات
6. اضغط "جرّب حظك" على أي محل
7. انتظر اللعبة (2-3 ثواني)
8. إما تفوز أو تحصل على خصم ترضية

### الخطوة 4: عرض الإحصائيات

1. ارجع للوحة المعلن `?admin=true`
2. شاهد الإحصائيات المحدثة:
   - مرات اللعب
   - الفائزين
   - معدل الفوز

## القواعد الأساسية

### محاولة واحدة يومياً
- يمكن للمستخدم اللعب مرة واحدة يومياً **لكل محل**
- التحقق يتم عبر `game_attempts` table
- الحساب بناءً على `CURRENT_DATE`

### Reward Engine
نظام الجوائز يعتمد على:
- `win_probability` من جدول campaigns
- توفر الجوائز (`quantity_remaining`)
- إذا لم تتوفر جوائز، يتحول إلى خسارة تلقائياً

### جائزة الترضية
كل من لا يفوز يحصل على:
- `consolation_prize` (وصف نصي)
- `consolation_discount` (قيمة رقمية)
- مثال: "خصم 2 ريال"

### صلاحية الجوائز
- 48 ساعة افتراضياً
- يتم حساب `expires_at` تلقائياً
- الحالات: pending → redeemed / expired

## API Endpoints

### الحصول على الحملات
```typescript
const { data } = await supabase
  .from('campaigns')
  .select(`
    *,
    neighborhoods(*),
    prizes(*)
  `)
  .eq('status', 'active')
  .eq('neighborhood_id', neighborhoodId);
```

### اللعب
```typescript
const { data } = await supabase.rpc('play_game', {
  p_user_id: userId,
  p_campaign_id: campaignId
});
```

### الحصول على تفاصيل الجائزة
```typescript
const { data } = await supabase
  .from('prize_claims')
  .select(`
    *,
    prizes(*),
    campaigns(*)
  `)
  .eq('claim_code', code)
  .eq('user_id', userId)
  .single();
```

- يمكن للمستخدمين رؤية محاولاتهم فقط

## المشاكل الشائعة

### "لقد استنفذت محاولاتك اليومية"
- تحقق من `game_attempts` table
- المحاولات تُحسب لليوم الحالي فقط
- يمكن اللعب مرة أخرى في اليوم التالي

### "لا توجد جوائز متاحة"
- تحقق من `prizes.quantity_remaining`
- قد تكون الجوائز نفذت
- قم بتحديث الكمية أو خفض نسبة الفوز

### OTP لا يظهر
- افتح Console (F12)
- ابحث عن "Verification code:"
- انسخ الكود والصقه
